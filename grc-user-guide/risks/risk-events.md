# GRC Risk Events
## Creating a Risk Event

* **Summary**

Write a summary of the risk event that occurred

* **Category**

Select a category that describes the risk event.

* **Details**

Provide the details of the risk event

* **Status**

Select the status of this risk event. Default options include Canceled, Closed, Investigating, Logged, Resolved, and Reviewing. (This list can be modified by a user with the GRC Administrator Role under GRC Simple Lists)

* **Priority**

Select the priority for this risk event. Default options include High, Immediate, Low, and Normal. (This list can be modified by a user with the GRC Administrator Role under GRC Simple Lists)

* **External Reference**

Free text field to record an external reference number

### Risk Occurrence

* **Register**

Select the Risk Register that this relates to

* **Risk**

Select the Risk that this event relates to. A Risk Register must be selected first, in order to display the list of available risks.

* **Asset Source**

Available Assets sources include GRC defined assets and Service Manager assets

* **Affected Asset**

Select the Asset that is affected by this Risk Event. Assets are defined within the GRC Configuration under the Framework and Asset Management section
